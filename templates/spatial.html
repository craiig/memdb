Spatial Locality Report

How to read these images:

Spatial Locality visualizations are meant to give a notion of how well each function call uses data that is adjacent in memory (as well as temporal locality, i.e. how well subsequent functions use the same memory). Spatial locality can be visualized as vertical or horizontal lines in these images.

The first image shows the current spatial locality in the original program. The second image shows spatial locality if one is able to re-order function calls for better spatial locality (i.e. we can reschedule function calls). This tool sorts by the memory addresses accessed in order by each function. If the sorted function image has significantly improved locality (more vertical lines), then the program will likely benefit from re-ordering the function calls. However, it remains a programming challenge to find the right parameter to sort by, because sorting by addresses accessed is usually infeasible.

  • y-axis: Different function invocations (Each row is a function call)
  • x-axis: Addresses in the order they are allocated in memory. (Adjacent columns are adjacent in memory)
  • Black dots are reads, Red dots are writes
